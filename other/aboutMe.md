## whoami

I'm **J4xx3n** — an independent security researcher and IT/security professional based in Madison, Wisconsin. I work across both sides of the fence: offensive security research on the bug bounty side, and defensive network security infrastructure on the day job.

This site is my personal knowledge base — a living collection of notes, methodology, custom tooling, and lessons learned from the field.

## What I Do

**Offensive security / bug bounty**
I've spent the last few years doing bug bounty work on **HackerOne** and **Bugcrowd** under the handle `J4xx3n`. A lot of my research centers on web application security — open redirects, XSS, CSP misconfigurations, SSRF, CRLF injection, nginx misconfigurations (alias traversal, ACL bypass, normalization issues), and CVE-based testing.

**Defensive / infrastructure**
On the operational side, I manage network security infrastructure — threat intelligence automation, EDL blocklist pipelines, and the plumbing that keeps a network's perimeter honest. Cross-platform IT support (Linux, Windows, WSL) is a recurring part of the work too.

## Tooling & Ecosystem

I build most of my recon and testing automation around the **ProjectDiscovery** stack:

- `nuclei` — I maintain a custom template library organized by vulnerability class
- `httpx`, `subfinder`, `katana`, `asnmap`, `mapcidr`, `cvemap`
- Wayback Machine (CDX API) and certificate transparency for recon
- Custom Python tooling for filtering, fuzzing, and aggregation

Most of this runs on Linux (Kali, Ubuntu, RHEL), often inside a Proxmox homelab where I spin up vulnerable/secured lab environments for testing.
